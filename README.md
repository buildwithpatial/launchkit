# launchkit

An opinionated Next.js 16 starter — auth, payments, and a typed AI client out of the box. Spin up a new project in 30 seconds, fill in the env vars, ship.

## What's in it

- **Auth** — NextAuth v5 with Credentials, Google, Apple, and guest sessions. Postgres-backed via Drizzle.
- **Payments** — Razorpay one-time checkout with HMAC signature verification + webhook safety net.
- **AI** — A typed `ai` client (`lib/launchkit/ai.ts`) wrapping `@ai-sdk/google` for `streamText` / `generateImage`. Defaults to Gemini.
- **Chat** — Full chat UI at `/chat` (model picker, history, attachments, artifacts) — keep it or delete it.
- **Demo home** — Live tiles on `/` showing each capability calling its own API route.

Locked stack: Next.js 16 + React 19 + Postgres (Neon) + Redis + Vercel Blob + Razorpay. No feature toggles — everything ships dormant by default; you wire in what you need.

## Use this template

1. On GitHub, click **Use this template** → **Create a new repository**, or run:
   ```bash
   gh repo create my-app --template buildwithpatial/launchkit --private
   ```
2. Clone and initialize:
   ```bash
   gh repo clone <your-org>/my-app
   cd my-app
   pnpm install
   node bin/init.mjs
   ```
   The init script renames `launchkit` → `my-app` in the right places, generates a fresh `AUTH_SECRET`, writes `.env.local`, then deletes itself.
3. Fill in `.env.local`. Eight values still need real keys — links in `.env.example`.
4. Run:
   ```bash
   pnpm db:migrate    # after POSTGRES_URL is set
   pnpm dev
   ```

Your app is at [localhost:3000](http://localhost:3000).

## Required services

| Var | Where to get it |
|---|---|
| `POSTGRES_URL` | https://neon.tech (free tier) |
| `AI_GATEWAY_API_KEY` | https://vercel.com/ai-gateway |
| `BLOB_READ_WRITE_TOKEN` | https://vercel.com/docs/vercel-blob |
| `REDIS_URL` | https://vercel.com/docs/redis |
| `AUTH_GOOGLE_ID` / `_SECRET` | https://console.cloud.google.com/apis/credentials |
| `AUTH_APPLE_ID` / `_SECRET` | https://developer.apple.com/account/resources/identifiers — JWT, see https://authjs.dev/getting-started/providers/apple |
| `RAZORPAY_KEY_ID` / `_SECRET` | https://dashboard.razorpay.com (test mode for development) |

`AUTH_SECRET` is auto-generated by the init script.

## Project layout

```
app/
├── (auth)/        # login, register, OAuth callbacks
├── (chat)/        # chat UI + APIs (delete if unused)
├── api/
│   ├── ai/        # text/image/chat endpoints powering the demos
│   └── payments/  # /order, /verify, /webhook
├── layout.tsx     # root
└── page.tsx       # demo home

components/
├── chat/          # chat UI primitives
├── launchkit/     # demo tiles + branded namespace (do not rename)
└── ui/            # shadcn primitives

lib/
├── ai/            # provider wiring, entitlements
├── db/            # Drizzle schema + queries + migrations
├── launchkit/ai.ts  # typed AI client (do not rename)
└── razorpay.ts    # server SDK + signature verifiers
```
