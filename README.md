# launchkit

An opinionated Next.js 16 starter ‚Äî auth, payments, and a typed AI client out of the box. Spin up a new project in 30 seconds, fill in the env vars, ship.

## What's in it

- **Auth** ‚Äî NextAuth v5 with Credentials, Google, and guest sessions. Postgres-backed via Drizzle.
- **Payments** ‚Äî Razorpay one-time checkout with HMAC signature verification + webhook safety net.
- **AI** ‚Äî A typed `ai` client (`lib/launchkit/ai.ts`) wrapping `@ai-sdk/google` for `streamText` / `generateImage`. Defaults to Gemini.
- **Chat** ‚Äî Full chat UI at `/chat` (model picker, history, attachments, artifacts) ‚Äî keep it or delete it.
- **Demo home** ‚Äî Live tiles on `/` showing each capability calling its own API route.

Locked stack: Next.js 16 + React 19 + Postgres (Neon) + Redis + Vercel Blob + Razorpay. No feature toggles ‚Äî everything ships dormant by default; you wire in what you need.

## Use this template

1. On GitHub, click **Use this template** ‚Üí **Create a new repository**, or run:
   ```bash
   gh repo create my-app --template buildwithpatial/launchkit --private
   ```
2. Clone and initialize:
   ```bash
   gh repo clone <your-org>/my-app
   cd my-app
   pnpm install
   node bin/init.mjs
   ```
   The init script renames `launchkit` ‚Üí `my-app` in the right places, generates a fresh `AUTH_SECRET`, writes `.env.local`, then deletes itself.
3. Fill in `.env.local`. Eight values still need real keys ‚Äî links in `.env.example`.
4. Run:
   ```bash
   pnpm db:migrate    # after POSTGRES_URL is set
   pnpm dev
   ```

Your app is at [localhost:3000](http://localhost:3000).

## Required services

| Var | Used for | Where to get it |
|---|---|---|
| `POSTGRES_URL` | User accounts, chat history, payments ‚Äî everything DB-backed | https://vercel.com/docs/postgres (provisions a Neon database) |
| `GOOGLE_GENERATIVE_AI_API_KEY` | Chat, text demo, image demo (Gemini) | https://aistudio.google.com/apikey |
| `BLOB_READ_WRITE_TOKEN` | File uploads in chat (images, PDFs as attachments) | https://vercel.com/docs/vercel-blob |
| `REDIS_URL` | Per-IP rate limiting on AI endpoints | https://vercel.com/docs/redis |
| `AUTH_GOOGLE_ID` / `_SECRET` | "Sign in with Google" button on login/register | https://console.cloud.google.com/apis/credentials |
| `RAZORPAY_KEY_ID` / `_SECRET` | Server-side order creation + signature verification | https://dashboard.razorpay.com (test mode for development) |
| `NEXT_PUBLIC_RAZORPAY_KEY_ID` | Client-side checkout modal (same value as `RAZORPAY_KEY_ID`) | (copy from above) |
| `RAZORPAY_WEBHOOK_SECRET` | Webhook signature verification (production safety net ‚Äî set up after deploy) | https://dashboard.razorpay.com/app/webhooks |

`AUTH_SECRET` is auto-generated by the init script. Email/password auth works without any of the OAuth keys.
